7.4 — failure analysis (qualitative)
recovered tool failure
tool error → split-phase recovery
Turn 3, task 14. Agent called memory({ action:"updateProfile", updates:{...} }) without userId. Runtime caught the missing parameter, forced re-entry to Thought Phase. Agent reformulated with correct parameter structure. Second attempt succeeded. User received no visible error. Recovery added ~340ms latency and one additional thought step.
retrieval miss
semantic retrieval miss at low cosine similarity
Task 31. User asked "what were those numbers we went over?" — low-specificity query produced top cosine score of 0.62, below the 0.70 threshold. dbsearch returned empty. Agent responded it could not find the referenced data and asked for clarification. The relevant message existed in the archive but the query embedding was too vague for reliable retrieval. Reducing minScore to 0.55 recovered the message but introduced two false positives in separate tests. Precision–recall tradeoff remains an open tuning problem.
trimming event
context trimming mid-task
Task 47, turn 18. Active context hit 60% threshold mid-scratchpad. System popped the oldest completed turn (turns 1–2), which contained the user's original task framing. Agent continued correctly because the Scratchpad captured task state and goals persisted in UserProfile. No user-visible disruption. Trimmed messages remained accessible via dbsearch for the remainder of the session.
architectural limit
failure despite architecture — plan error on ambiguous task
Task 62. User: "Handle the usual end-of-week thing." No prior session context. Agent entered Thought Phase, produced a plausible but incorrect interpretation, and executed the wrong tool sequence. The split-phase loop did not help because the error was semantic, not mechanical — no tool call failed, so no recovery was triggered. The architecture recovers from tool-execution failures; it cannot recover from a fundamentally wrong plan when the task is underspecified. Clarification prompting is the correct mitigation, not architecture.
